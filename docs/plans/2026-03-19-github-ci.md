# GitHub CI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a minimal GitHub Actions CI pipeline for this repository that enforces formatting, linting, tests, and the existing `100%` coverage requirement without adding release automation or multi-version matrix complexity.

**Architecture:** The implementation stays intentionally small. Ruff becomes the only formatting and linting tool, GitHub Actions runs three focused jobs on Python `3.10`, and pytest keeps ownership of coverage enforcement through the existing `pytest.ini` settings.

**Tech Stack:** GitHub Actions, Python 3.10, Ruff, pytest, pytest-cov, Markdown

### Task 2: Add The GitHub Actions CI Workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Write the workflow**

Create `.github/workflows/ci.yml` with:

- workflow name `CI`
- triggers for `pull_request` and `push` to `master`
- `concurrency` keyed by workflow name and ref
- three jobs: `format`, `lint`, `test`
- shared setup in each job:
  - `actions/checkout@v4`
  - `actions/setup-python@v5` with `python-version: "3.10"`
  - `python -m pip install --upgrade pip`
  - `python -m pip install -r requirements-dev.txt`
- job commands:
  - `python -m ruff format --check .`
  - `python -m ruff check .`
  - `pytest -q`

**Step 2: Verify the workflow mirrors local CI commands**

Run:

```bash
python3.10 -m pip install -r requirements-dev.txt
python3.10 -m ruff format --check .
python3.10 -m ruff check .
python3.10 -m pytest -q
```

Expected: all commands PASS locally, proving the workflow steps are valid for the repository.

**Step 3: Commit the workflow**

Run:

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions CI workflow"
```

Expected: one commit adding only the workflow file.
